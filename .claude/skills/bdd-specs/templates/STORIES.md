# User Stories

> Required by the `bdd-specs` skill. Put this at `/docs/STORIES.md`.
> Each story becomes one `Feature`. Each acceptance criterion becomes
> one `Scenario`. Each expected outcome becomes one `it` (one assertion).

## Story 1: <short title>

**As a** <role>
**I want** <capability>
**So that** <value / outcome>

### Acceptance criteria (happy path — be exhaustive)

- **Given** <starting state> **when** <action> **then** <observable outcome>
- **Given** … **when** … **then** …

### Failure / edge cases (sad path — segregated into their own scenarios)

- **Given** <state> **when** <invalid action> **then** <rejection / error>
- **Given** … **when** … **then** …
